Output Formats

Vibetest DAQ can write acquired data as CSV (text) or HDF5 (binary). Both carry the same data and metadata.

Choosing a Format

CSVHDF5
Extension .csv .h5
Type Plain text Binary
Human-readable Yes — open in any text editor or spreadsheet No — requires a viewer or code to inspect
Read speed Slower (text parsing) Much faster (binary float64 directly)
File size Larger (text encoding) Smaller (roughly 40–60% of CSV)
Metadata Comment lines (#) in the file header HDF5 attributes (structured, queryable)
Best for Quick inspection, simple tools, archival Real-time readback by vibetest-analyzer, large datasets
When to use HDF5: If vibetest-analyzer is reading and analyzing files in real time as they are generated, HDF5 is strongly recommended. The binary format eliminates text-parsing overhead, so the analyzer can load each new file almost instantly. For post-hoc analysis where speed is less critical, CSV is convenient because you can inspect files with any text editor.

Select the format in the GUI with the Output format combo box on the Acquire tab, or in the CLI with the --format csv|hdf5 flag.

File Naming

Both formats use the same naming pattern, differing only in extension:

<prefix>_YYYYMMDD_HHMMSS_mmm.csv
<prefix>_YYYYMMDD_HHMMSS_mmm.h5

Timestamp Accuracy

Timestamps in both formats are computed from the acquisition start time plus a sample counter, not from wall-clock readings inside the acquisition loop. This eliminates per-block jitter caused by file-write latency and loop overhead. The formula is:

t = acquisition_start_utc + (block_number × samples_per_block + sample_index) / actual_sample_rate
Why epoch time? Unix epoch timestamps are unambiguous — they have no time-zone or daylight-saving-time complications. Most analysis tools (Python, MATLAB, R) can convert epoch seconds to human-readable dates easily.

CSV Format Details

Each CSV file has two sections: a comment header (lines starting with #) and the data rows.

Comment Header

Header lines carry metadata about the acquisition. Most CSV parsers skip lines starting with # automatically.

# NI cDAQ-9174 / NI 9230 Vibration Data
# Block start (UTC): 2026-05-20T18:30:00
# Block start (epoch s): 1779316200.000000
# Sample rate (Hz):  5000.0
# Samples:           50000
# Channels:          6
# Sensitivity (mV/g):100.0
# Units:             g (acceleration)
# Test ID: FAN-SWEEP-001
# DUT Make: Acme
# DUT Model: VX-10
# DUT Serial Number: SN-0042
# Test Stand: Stand A
# Operator: J. Smith
# Location: Lab 3
# Test Notes: Baseline run, no load
# Channel Mod1_Ch0 Axis: X
# Channel Mod1_Ch0 Location: Top plate
# Channel Mod1_Ch0 Sensor Serial: PCB-12345
# --------------------------------------------------------------
time_epoch_s,Mod1_Ch0,Mod1_Ch1,Mod1_Ch2,Mod2_Ch0,Mod2_Ch1,Mod2_Ch2

System and channel metadata lines appear only when those fields were filled in. See Metadata and Channels for details.

Column Header

The last line before the data is the column header (not prefixed with #). Only enabled channels appear:

time_epoch_s,Mod1_Ch0,Mod1_Ch1,Mod1_Ch2,Mod2_Ch0,Mod2_Ch1,Mod2_Ch2

Data Rows

1779316200.000000,0.0012345,-0.0023456,0.0034567,0.0011234,-0.0022345,0.0033456
1779316200.000200,0.0012678,-0.0023789,0.0034890,0.0011567,-0.0022678,0.0033789
ColumnFormatDescription
time_epoch_s 6 decimal places Absolute UTC time as Unix epoch seconds. Each row increments by 1/sample_rate.
Channel columns Up to 8 significant digits Acceleration in g. The NI-DAQmx driver applies the sensitivity calibration, so these are engineering units, not raw voltages.

Loading CSV Files

Python (NumPy)

import numpy as np

data = np.loadtxt("vib_20260520_183000_000.csv",
                  delimiter=",", comments="#", skiprows=0)
time_s  = data[:, 0]    # epoch seconds
ch0     = data[:, 1]    # Mod1_Ch0 in g

Python (pandas)

import pandas as pd

df = pd.read_csv("vib_20260520_183000_000.csv", comment="#")
# Columns: time_epoch_s, Mod1_Ch0, Mod1_Ch1, ...

MATLAB

data = readmatrix('vib_20260520_183000_000.csv', ...
    'CommentStyle', '#');
t = data(:, 1);     % epoch seconds
ch0 = data(:, 2);   % Mod1_Ch0 in g

HDF5 Format Details

Each HDF5 file contains two datasets and a set of attributes that carry the same information as the CSV comment header. Because the data is stored as binary float64 arrays, there is no text parsing on read — the arrays map directly into memory.

File Structure

file.h5
├── root attributes
│   ├── block_start_utc       "2026-05-20T18:30:00"
│   ├── block_start_epoch_s   1779316200.0
│   ├── sample_rate_hz        5000.0
│   ├── n_samples             50000
│   ├── n_channels            6
│   ├── sensitivity_mv_per_g  100.0
│   ├── units                 "g"
│   ├── channel_labels        ["Mod1_Ch0", "Mod1_Ch1", ...]
│   ├── test_id               "FAN-SWEEP-001"        (if set)
│   ├── dut_make              "Acme"                 (if set)
│   ├── dut_model             "VX-10"                (if set)
│   ├── dut_serial            "SN-0042"              (if set)
│   ├── test_stand            "Stand A"              (if set)
│   ├── operator              "J. Smith"             (if set)
│   ├── location              "Lab 3"                (if set)
│   └── notes                 "Baseline run, no load" (if set)
│
├── time_epoch_s              dataset  (n_samples,)       float64
│
└── data                      dataset  (n_samples, n_channels)  float64
    └── attributes
        ├── Mod1_Ch0_axis            "X"           (if set)
        ├── Mod1_Ch0_location        "Top plate"   (if set)
        ├── Mod1_Ch0_sensor_serial   "PCB-12345"   (if set)
        └── ...                      (per enabled channel)

Root Attributes

AttributeTypeDescription
block_start_utcstringISO 8601 timestamp of the first sample
block_start_epoch_sfloat64Same timestamp as Unix epoch seconds
sample_rate_hzfloat64Actual sample rate used by the hardware
n_samplesintNumber of samples per channel
n_channelsintNumber of channels in the data dataset
sensitivity_mv_per_gfloat64Accelerometer sensitivity
unitsstringEngineering units ("g")
channel_labelsstring arrayOrdered list of channel names matching the columns of the data dataset

System metadata fields (test_id, dut_make, dut_model, dut_serial, test_stand, operator, location, notes) are stored as additional root attributes when non-empty.

Datasets

DatasetShapedtypeDescription
time_epoch_s (n_samples,) float64 Absolute UTC time for each sample as Unix epoch seconds
data (n_samples, n_channels) float64 Acceleration readings in g. Column order matches channel_labels.

Per-channel metadata (axis, location, sensor serial) is stored as attributes on the data dataset, keyed as <channel_label>_<field> (e.g., Mod1_Ch0_axis).

Selective reads: To read a single channel without loading the entire dataset, use HDF5 hyperslab selection: f["data"][:, 0] reads only the first channel column.

Loading HDF5 Files

Python (h5py)

import h5py

with h5py.File("vib_20260520_183000_000.h5", "r") as f:
    fs       = f.attrs["sample_rate_hz"]
    labels   = list(f.attrs["channel_labels"])
    time_s   = f["time_epoch_s"][:]      # (n_samples,)
    all_data = f["data"][:]              # (n_samples, n_channels)
    ch0      = f["data"][:, 0]           # single channel

Python (h5py — reading metadata)

with h5py.File("vib_20260520_183000_000.h5", "r") as f:
    test_id = f.attrs.get("test_id", "")
    dut     = f"{f.attrs.get('dut_make', '')} {f.attrs.get('dut_model', '')}"
    ch0_axis = f["data"].attrs.get("Mod1_Ch0_axis", "")

MATLAB

info = h5info('vib_20260520_183000_000.h5');

File Size Estimates

The table below gives rough file sizes for common configurations. HDF5 files are smaller because they store float64 values in binary (8 bytes each) rather than as variable-length text.

Sample rateBlock durationChannelsCSVHDF5
5,000 Hz1 s6~500 KB~280 KB
5,000 Hz10 s6~5 MB~2.7 MB
12,800 Hz10 s6~13 MB~7 MB
5,000 Hz10 s3~3 MB~1.5 MB
Disk space: Long continuous runs at high sample rates generate large volumes of data. For a 5 kHz, 6-channel acquisition using CSV, expect roughly 1.7 GB per hour. HDF5 cuts this to approximately 1 GB per hour. Monitor available disk space for long acquisitions.
