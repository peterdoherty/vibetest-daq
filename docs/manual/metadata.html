Metadata

The Metadata tab lets you document your test setup so every CSV file carries full context.

Why Metadata Matters

When you look at a vibration CSV file weeks or months later, the raw numbers alone do not tell you what was being tested, how the sensors were mounted, or who ran the test. The Metadata tab captures this context at acquisition time and writes it directly into the CSV file headers. This makes every file self-describing and eliminates the need for separate log sheets.

System Metadata Fields

The Metadata tab contains the following fields. All are optional, but filling them in is strongly recommended.

FieldPurposeExample
Test ID A unique identifier for this test or test series. Use whatever naming convention your team follows. FAN-SWEEP-001
DUT make Manufacturer of the device under test. Acme
DUT model Model number or name of the device under test. VX-10
DUT serial Serial number of the specific unit being tested. SN-0042
Test stand Which test fixture or stand the DUT is mounted on. Stand A
Operator Name of the person running the test. J. Smith
Location Physical location where the test is taking place. Lab 3, Building 60
Notes Free-form text for anything else — test intent, fixture configuration, environmental conditions, known issues, etc. Multi-line text is supported. Baseline run, no load, 22°C ambient

How Metadata Appears in CSV Files

Each non-empty metadata field is written as a comment line in the CSV header. For example:

# NI cDAQ-9174 / NI 9230 Vibration Data
# Block start (UTC): 2026-05-20T18:30:00
# Sample rate (Hz):  5000.0
# Channels:          6
# Sensitivity (mV/g):100.0
# Units:             g (acceleration)
# Test ID: FAN-SWEEP-001
# DUT Make: Acme
# DUT Model: VX-10
# DUT Serial Number: SN-0042
# Test Stand: Stand A
# Operator: J. Smith
# Location: Lab 3, Building 60
# Test Notes: Baseline run, no load | 22C ambient
Note: Multi-line notes are collapsed to a single line in the CSV header, with line breaks replaced by | pipe characters. This keeps the CSV file easy to parse while preserving the full note text.

Metadata Summary on the Acquire Tab

You do not need to switch to the Metadata tab to verify your entries. The Metadata Summary panel on the Acquire tab updates in real time as you type. It shows a compact view of the test ID, DUT, setup details, active channels, and notes — so you can confirm everything looks right before pressing Start.

Long notes are truncated to 120 characters in the summary display, but the full text is always written to the CSV files.

Persistence

All metadata fields are saved automatically when you start an acquisition or close the application. The next time you open the GUI, the fields are pre-filled with their previous values. This is convenient when running multiple tests on the same DUT — just update the fields that changed (such as the test ID or notes) and leave the rest.

Remember to update metadata between tests. Because fields persist, it is easy to accidentally record a new test with the previous test's metadata. Get in the habit of reviewing the Metadata Summary on the Acquire tab before each run.
