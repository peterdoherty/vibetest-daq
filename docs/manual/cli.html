Command Line Interface

Run acquisitions from a terminal for scripted, automated, or unattended recording.

When to Use the CLI

The command-line interface (vibetest-daq) is a simpler alternative to the GUI. It records data from all six channels using a fixed configuration and writes CSV files to disk until you stop it. Use it when:

  • You want to run an acquisition from a script or batch file.
  • You need unattended recording on a headless machine (no display).
  • You prefer the simplicity of a terminal over a graphical interface.
Note: The CLI always records all six channels and does not support metadata fields or per-channel labels. If you need those features, use the GUI.

Basic Usage

vibetest-daq [OPTIONS]

Options

OptionDefaultDescription
-d, --duration SECONDS None (run forever) Stop recording after this many seconds. If omitted, the acquisition runs until you press Ctrl+C.
-o, --output DIR vibration_data Directory where CSV files are written. Created automatically if it does not exist.
-r, --rate HZ 5000 Requested sample rate in Hertz.
-f, --format csv|hdf5 csv Output file format. Use hdf5 for faster readback by downstream analysis tools. See Output Formats for details on both formats.

Examples

Record for 60 seconds at 5 kHz

vibetest-daq --duration 60 --output ./test_data --rate 5000

This creates approximately 60 CSV files (one per second, since the default block duration is 1 s) in the ./test_data directory.

Record indefinitely until interrupted

vibetest-daq --output ./long_run

Press Ctrl+C to stop. The acquisition halts cleanly after the current block finishes writing.

Record at a higher sample rate

vibetest-daq --rate 12800 --output ./highrate_data

Record in HDF5 format

vibetest-daq --format hdf5 --output ./test_data

Output files will have a .h5 extension instead of .csv. The data content is identical; only the container format differs.

Logging

The CLI prints log messages to the terminal as it runs. You will see messages for:

  • The configured output directory, sample rate, and block size.
  • The actual sample rate achieved by the hardware (with a warning if it differs significantly from the requested rate).
  • Each CSV file written, with its path and dimensions.
  • Any DAQ errors that occur during recording.
