What is Vibetest DAQ?

Vibetest DAQ is a data acquisition tool that records vibration measurements from IEPE (Integrated Electronics Piezo-Electric) accelerometers connected to National Instruments hardware. It saves each block of samples as a timestamped CSV file that you can later analyze with spectral analysis software, MATLAB, Python, or any tool that reads plain text.

The package provides two ways to run an acquisition:

  • GUI (vibetest-daq-gui) — a desktop application with controls for all acquisition settings, test metadata, per-channel configuration, and live level meters.
  • Command line (vibetest-daq) — a headless script suitable for automated or unattended recording.

Key Concepts

IEPE Sensors

IEPE accelerometers contain a built-in charge amplifier powered by a constant-current excitation (typically 4 mA). The NI 9230 supplies this current and reads the resulting AC voltage. The sensitivity of the sensor (in mV/g) converts that voltage to acceleration in units of g (1 g = 9.81 m/s²).

Blocks and Files

Data is acquired continuously, but it is written to disk in fixed-length blocks. Each block becomes one CSV file. The block duration setting controls how many seconds of data go into each file. For example, at a 5 kHz sample rate with a 10-second block duration, each file contains 50,000 samples per channel.

Sample Rate

The sample rate determines how many readings per second the hardware takes on each channel. To capture vibrations up to a frequency f, you need a sample rate of at least 2f (the Nyquist criterion). A 5 kHz rate lets you analyze frequencies up to 2.5 kHz.

Note: The hardware may adjust the requested sample rate slightly to match its internal clock dividers. The actual rate used is reported in both the GUI status bar and the CSV file header.
