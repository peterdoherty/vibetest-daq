GUI Overview

The Acquire tab is the primary operational screen — your cockpit for running vibration tests.

Home Getting Started GUI Overview Metadata Channels Command Line Output Formats Troubleshooting

Window Layout

The GUI window is organized into three tabs across the top:

  1. Acquire — acquisition settings, start/stop controls, live status, and channel level meters (this page).
  2. Metadata — test identity, device-under-test info, and operator details (see Metadata).
  3. Channels — per-channel enable/disable, axis labels, sensor locations, and serial numbers (see Channels).

A status bar at the bottom of the window shows real-time messages about the current state of the system.

Acquisition Settings

The top section of the Acquire tab contains all of the parameters that control how data is captured and where it is saved.

SettingDefaultDescription
Output directory vibration_data Folder where CSV files are written. Use the Browse button to choose a location. The directory is created automatically if it does not exist.
File prefix vib Text prepended to each output filename. Files are named <prefix>_YYYYMMDD_HHMMSS_mmm.csv (or .h5 for HDF5).
Output format CSV File format for acquired data. CSV produces human-readable text files that can be opened in any editor or spreadsheet. HDF5 produces compact binary files that are much faster to read back programmatically — useful when vibetest-analyzer is processing files in real time as they are generated. See Output Formats for details on both.
Requested rate 5000 Hz Desired sample rate in Hertz. Valid range: 1,000–51,200 Hz. The hardware may use a slightly different rate; the actual rate is shown in the Status section once recording begins.
Block duration 10 s Length of each data block in seconds. Each block produces one CSV file. Valid range: 1–3,600 s.
Sensitivity 100 mV/g Accelerometer sensitivity. Check the datasheet for your sensors and set this to match.
IEPE excitation 0.004 A (4 mA) Constant-current excitation supplied to the IEPE sensors. The NI 9230 typically accepts 0 A (off) or 0.004 A.
Module 1 / Module 2 device cDAQ1Mod1 / cDAQ1Mod2 NI-DAQmx device names for the two NI 9230 modules. These must match what NI MAX reports.
File count 10 Number of files (blocks) to record when Continuous is unchecked. After this many files are written, acquisition stops automatically.
Continuous Checked When checked, acquisition runs indefinitely until you press Stop. When unchecked, it stops after the specified file count.
All settings are locked while recording. You must stop the current acquisition before changing any parameter. This prevents accidental mid-run configuration changes that could produce inconsistent data.

Metadata Summary

Below the acquisition settings is a read-only Metadata Summary panel. It shows a compact preview of the current test identity, DUT information, active channels, and notes — so you can confirm the metadata is correct without switching to the Metadata tab.

Example summary:
Test: FAN-SWEEP-001
DUT: Acme VX-10 SN-42
Setup: Stand A | J. Smith | Lab 3
Channels: Mod1_Ch0=Top X; Mod1_Ch1=Top Y; Mod1_Ch2=Top Z
Notes: Baseline run, no load

Start and Stop

The two transport buttons control the acquisition lifecycle:

  • Start Acquisition — validates that at least one channel is enabled, saves all current settings, then connects to the DAQ hardware and begins recording. The button is disabled while recording.
  • Stop — signals the acquisition to halt after the current block finishes writing. Enabled only while recording.

Acquisition runs in a background thread, so the GUI remains responsive while data is being captured.

Channel Level Meters

At the bottom of the Acquire tab, six horizontal bar meters show the peak acceleration (in g) for each channel during each block. The meters update once per block.

Color coding

ColorMeaning
GreenPeak is below 50% of full scale
AmberPeak is between 50% and 80% of full scale
RedPeak exceeds 80% of full scale — the sensor may be close to clipping

Full scale control

The Full scale spinner above the meters sets the reference level (in g) used for the color thresholds. Adjust it to match the expected vibration amplitude of your test. For example, if you expect peaks around 0.5 g, set full scale to 1.0 g so the meters use the full bar range.

Note: Disabled channels show "off" instead of a meter bar. Only channels enabled on the Channels tab display live readings.

Settings Persistence

All GUI settings — acquisition parameters, metadata fields, and channel configuration — are saved automatically when you press Start or close the window. The next time you open the application, your previous settings are restored. This is handled by Qt's QSettings system, which stores values in a platform-appropriate location (the Windows registry, macOS preferences, or an INI file on Linux).
