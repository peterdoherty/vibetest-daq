The Channels Tab

The Channels tab shows a table with one row per channel. Each row has the following columns:

ColumnDescription
Enable A checkbox that includes or excludes this channel from the acquisition. When unchecked, the channel is not configured on the hardware, no data is recorded for it, and its level meter on the Acquire tab shows "off".
Channel The fixed hardware channel name (e.g., Mod1_Ch0). This is read-only.
Axis The measurement axis label for this channel. Defaults to X, Y, or Z. You can change this to match your sensor orientation or use any short label you prefer.
Location A free-text field describing where on the DUT this sensor is mounted. For example: Top plate, Base flange, Motor housing.
Sensor serial The serial number of the physical accelerometer connected to this channel. Useful for traceability when you need to know which sensor produced a particular measurement.

How Channel Metadata Appears in CSV Files

Per-channel metadata is written into the CSV header as comment lines. Only non-empty fields are included:

# Channel Mod1_Ch0 Axis: X
# Channel Mod1_Ch0 Location: Top plate
# Channel Mod1_Ch0 Sensor Serial: PCB-12345
# Channel Mod1_Ch1 Axis: Y
# Channel Mod1_Ch1 Location: Top plate
# Channel Mod1_Ch1 Sensor Serial: PCB-12346

Downstream analysis software can parse these headers to label plots as "Top plate X" instead of the generic "Mod1_Ch0".

Enabling and Disabling Channels

You do not always need all six channels. If a test only uses three sensors (one triaxial accelerometer, for example), uncheck the unused channels. This has several benefits:

  • The CSV files are smaller because unused columns are omitted.
  • The NI-DAQmx task only configures the channels it needs, which can reduce bus bandwidth.
  • The level meters on the Acquire tab only show active channels, reducing visual clutter.
At least one channel must be enabled. If you try to start an acquisition with all channels disabled, the application will show a warning and refuse to start.

Channel Summary

The Metadata Summary on the Acquire tab includes a Channels line that shows a compact view of enabled channels and their axis/location labels:

Example: Channels: Mod1_Ch0=Top plate X; Mod1_Ch1=Top plate Y; Mod1_Ch2=Top plate Z

Persistence

Channel enable states, axis labels, location descriptions, and sensor serial numbers are all saved automatically. When you reopen the application, each channel is restored to its previous configuration.
